# 前端目录分层初始化

## 修改时间

2026-07-13 17:19:03 CST

## 修改原因

项目需要在正式开发具体功能前，先建立稳定、可检查的前端代码分层，并为后续每次前端代码和逻辑变更提供统一记录位置。

## 修改范围

- 创建 `frontend/` 前端根目录；
- 建立 Next.js 预期代码分层；
- 创建前端修改记录目录；
- 本次不初始化 Next.js，不安装依赖，不编写业务代码。

## 涉及目录

```text
frontend/
├── change-records/
├── public/
├── src/
│   ├── app/
│   ├── components/
│   │   ├── ui/
│   │   └── shared/
│   ├── features/
│   ├── hooks/
│   ├── lib/
│   ├── services/
│   ├── stores/
│   ├── styles/
│   └── types/
└── tests/
    └── e2e/
```

## 实际修改

- `app` 用于 Next.js 路由、布局和页面入口；
- `features` 用于按后续实际业务功能组织代码；
- `components/ui` 用于基础 UI 组件；
- `components/shared` 用于跨功能复用组件；
- `services` 用于后端 API 调用与数据适配；
- `stores` 仅用于必要的跨页面客户端状态；
- `tests/e2e` 用于端到端用户流程测试；
- 空目录使用 `.gitkeep` 保留，后续创建实际文件时可移除对应占位文件。

本次没有业务逻辑变化。新增的是前端代码的存放边界和依赖约束。
