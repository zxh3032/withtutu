# 后端目录分层初始化

## 修改时间

2026-07-13 17:19:03 CST

## 修改原因

项目已确定后端使用 Go、Gin、MySQL 和 Redis，需要在实现具体功能前建立清晰的后端分层，避免后续把路由、业务逻辑和数据访问混合在一起。

## 修改范围

- 创建 `backend/` 后端根目录；
- 建立 Go/Gin 预期代码分层；
- 为 MySQL、Redis 和 Migration 预留明确位置；
- 创建后端修改记录目录；
- 本次不初始化 Go Modules，不安装 Gin，不创建数据表，不编写业务代码。

## 实际修改

- `cmd/server` 用于 Go 服务启动入口；
- `transport/http` 用于 Gin 路由、Handler 和中间件；
- `application` 用于业务用例和流程编排；
- `domain` 用于领域实体、规则和数据访问接口；
- `infrastructure/persistence/mysql` 用于 MySQL 实现；
- `infrastructure/cache/redis` 用于 Redis 实现；
- `bootstrap` 用于配置依赖并组装应用；
- `migrations` 用于后续 MySQL 表结构变更；
- 空目录使用 `.gitkeep` 保留，后续创建实际文件时可移除对应占位文件。

## 逻辑变化

本次没有业务逻辑变化。新增后端依赖方向：

```text
transport → application → domain
                       ↑
              infrastructure
```

Gin、MySQL 和 Redis 的具体类型不得进入领域层。

## 数据库或缓存变化

- 未创建 MySQL 数据库和数据表；
- 未创建 Redis Key；
- 仅建立未来实现代码的目录位置。
