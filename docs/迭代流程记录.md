# withtutu 迭代流程记录

> 本文记录项目每次迭代的目标、范围、过程、结果和遗留问题。项目按照正常产品研发流程逐项实现已经明确的功能。

本文第 1～4 节是当前有效的迭代规范，属于正式正文；第 5～6 节是当前迭代和历史过程记录。迭代执行以第 1～4 节为准，历史内容用于追溯上下文。

## 1. 迭代原则

- 一次迭代聚焦一个明确目标或一组强相关变更；
- 开始开发前明确目标、范围和验收标准；
- 不因为长期可能需要某项能力就提前实现它；
- 前后端只建设当前功能实际需要的接口、数据和缓存；
- 迭代结束前完成测试、文档和修改记录；
- 未完成内容进入后续迭代，不以“临时先放着”代替记录；
- 所有迭代最终服务于完整产品，但不提前承诺尚未确认的功能清单。

## 2. 标准迭代流程

```text
提出功能或问题
→ 明确目标与用户价值
→ 确认范围和非范围
→ 完成产品与技术方案
→ 拆分前后端任务
→ 开发与测试
→ 记录实际代码和逻辑变更
→ 验收并关闭迭代
```

## 3. 迭代状态

| 状态 | 含义 |
|---|---|
| 待讨论 | 只有想法，目标和范围尚未确认 |
| 待开始 | 目标、范围和验收标准已经确认 |
| 进行中 | 已进入设计或开发 |
| 待验收 | 开发和验证已经完成，等待用户明确确认验收结果 |
| 已完成 | 用户已经明确确认验收通过，且实际变更和验证结果已经记录 |
| 已暂停 | 主动停止，需要说明原因和恢复条件 |

## 4. 迭代记录模板

```md
| 开始时间 | YYYY-MM-DD HH:mm |
| 完成时间 | YYYY-MM-DD HH:mm / — |

#### 迭代目标

说明本次要解决的问题和完成后的结果。

#### 本次范围

- 本次会完成的内容。

#### 非本次范围

- 本次明确不做的内容。

#### 方案与任务

- 产品或技术方案；
- 前端任务；
- 后端任务；
- 数据库和缓存任务；
- 测试与验收任务。

#### 实际变更

- 实际完成了什么；
- 与原计划有什么差异；
- 前后端修改记录文件链接。

#### 验证结果

- 执行的检查或测试；
- 验收结果；
- 已知问题。

#### 后续事项

- 未完成内容和下一步建议。
```
