# docs 目录总览

> 本文件记录 `docs/` 目录的整体结构、各文档用途和维护规则。新增、删除、移动或重命名文档时，必须同步更新本文件。

## 目录结构

```text
docs/
├── README.md
├── 技术选型说明.md
└── 迭代流程记录.md
```

## 文件说明

| 文件 | 用途 | 何时更新 |
|---|---|---|
| `README.md` | 记录文档目录结构和每份文档的职责 | 文档新增、删除、移动或改名时 |
| `技术选型说明.md` | 记录已经确认的前端、后端、数据库和缓存技术基线 | 技术选型被明确调整时 |
| `迭代流程记录.md` | 记录每次迭代的目标、范围、实际变更、验证结果和后续事项 | 开始、推进或结束一次迭代时 |

## 内容边界

`docs/` 用于存放跨前后端、需要长期维护的项目文档，例如：

- 技术选型；
- 迭代流程；
- 后续经过确认的跨端方案；
- 需要前后端共同遵守的设计说明。

以下内容不放在本目录：

- 前端具体代码修改记录，放在 `frontend/change-records/`；
- 后端具体代码修改记录，放在 `backend/change-records/`；
- 临时调试输出、构建产物和本地私人笔记；
- 尚未进入迭代、没有确认的功能设想。

## 维护规则

1. 新增文档前先确认它是否需要跨迭代长期维护；
2. 文档名称应准确表达内容，不使用“临时”“新建文档”等模糊名称；
3. 文档内容发生职责变化时，同时更新本文件的文件说明；
4. 删除文档前检查是否仍被 `README.md`、`AGENTS.md` 或迭代记录引用；
5. 目录结构变化必须在同一次修改中更新本文件；
6. 具体迭代结果应写入迭代记录，不在多个文档重复维护。
